Permission Modes

Normal
Claude asks permission for every tool use (read, write, bash, etc.). Full control.
Auto-Accept
Claude runs tools WITHOUT asking. Faster but less control. Good for trusted tasks.
Plan Mode
Claude ONLY reads & plans. Won't write or run anything. Review first, then switch to Normal to execute.
Shift+Tab→ Normal→ Auto-Accept→ Plan→ Normal…
Best workflow: Start in Plan Mode to explore & understand the problem. Review Claude's plan. Switch to Normal/Auto-Accept to implement.
✅

Fine-Grained Tool Control

Allow git only
Add "Bash(git:*)" to allowedTools in settings
Allow all bash
Add "Bash(*)" to allowedTools
Block writes
Add "Write" to disallowedTools
Read-only CLI
--tools "Read,Glob,Grep"
// .claude/settings.json { "allowedTools": ["Bash(git:*)", "Read"], "disallowedTools": ["Write"] }

How to Rewind

Esc ×2
Open rewind menu anywhere in the session
/rewind
Same as above but typed as a slash command
Conversation
Go back in chat only. Code files stay as-is.
Code
Restore files only. Conversation history stays.
Full Rewind
Restore both conversation AND code to a specific point.
Bash side-effects (database changes, API calls, files deleted via rm) can't be rewound. Checkpoints only track file edits by Claude. Use Git for permanent safety.
💾

Staying Safe

Auto-checkpoints
Rewind checkpoints are auto-created before each Claude action — you can always go back
Git safety net
Commit to git before Claude makes sweeping changes — permanent undo
Worktrees
claude -w feature — isolated branch means zero risk to main
Plan first
Use Plan Mode to review before any writes — prevents needing to rewind

Creating Custom Commands

1. Create file
.claude/commands/review.md
2. Write prompt
The markdown content IS the prompt Claude will use
3. Use it
Type /project:review in Claude Code
# .claude/commands/fix.md --- argument-hint: "filename or description" description: "Fix a bug in the specified file" allowed-tools: ["Read", "Write", "Bash"] --- Fix the bug: $ARGUMENTS Steps: 1. Read the relevant file 2. Identify the root cause 3. Write a failing test first 4. Fix the code and verify
📝

YAML Frontmatter Options

argument-hint
Placeholder text shown for argument input
description
Shows in /help listing
allowed-tools
Restrict what tools the command can use
model
Force a specific model for this command
$ARGUMENTS
Use $ARGUMENTS in your markdown — it gets replaced with whatever you type after the command name.
Example: /project:review src/auth.ts → $ARGUMENTS = "src/auth.ts"
