Install Git & GitHub CLI & Authenticate

Install Git, then GitHub CLI — run commands for your OS

›
ℹ️ Run these commands one at a time in your Terminal (Mac) or Command Prompt (Windows).

Step 1 — Install Git

⬇ git-scm.com/downloads
💻 Choose the installer for your operating system — Windows, Mac, or Linux — before downloading.
  • Select your OS (Windows / macOS / Linux) on the downloads page
  • Download and run the installer — click through all defaults
  • Restart your terminal when done
🪟 Windows users — extra step required. After installing Git, open Command Prompt and run these commands to set your global identity.
Command Prompt — Windows 
# Set your name
git config --global user.name "Your Name"

# Set your work email
git config --global user.email "your_email@example.com"

# Verify your settings
git config --global --list
✅ Replace "Your Name" and "your_email@example.com" with your @accessinfinity.com email. The last command confirms your settings are saved.

Step 2 — Install GitHub CLI & Authenticate

🍎 macOS

Terminal — macOS 
# Install Homebrew
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# Install GitHub CLI
brew install gh

# Log in to GitHub
gh auth login

🪟 Windows

Command Prompt — Windows 
# Install GitHub CLI using winget (built into Windows 10/11)
winget install --id GitHub.cli

# Restart Command Prompt, then log in to GitHub
gh auth login
⚠️ Windows users: After running winget install, close and reopen your Command Prompt before running gh auth login.
✅ When prompted by gh auth login — choose GitHub.com → HTTPS → Login with a web browser and follow the steps.

Daily Git & Firebase Workflow

Pull → push → deploy — do this every day

›
1
Get Latest Code
Ask your developer to pull the latest version of the code before any changes are made.
2
Push Latest Code
Once changes are ready, your developer will push the updated code to the main repository.
3
Deploy in Firebase
Your developer will deploy the latest code to Firebase. You'll receive a link to access the live app.
💡 Good habit: Always pull first → make changes → push → deploy. Never push without pulling first to avoid conflicts.

⬇ Get latest code from remote

Always start here — get the latest version from remote.

git pull origin main

⬆ Push the latest code to remote main

Stage, commit, and push your work.

git add .
git commit -m "message"
git push origin main

🚀 firebase deploy latest code

Ship the latest code live after pushing.

firebase deploy
